console.log() - базовый инструмент, чтобы видеть, что выводит что-либо;

переменные:

    const - для того, что не меняется (в т.ч. ссылка на кнопку в коде);

    let - для того, что будет меняться (счётчик, статы);

    var - использовать крайне выборочно. как минимум странная область
    видимости: видна в пределах функции, даже если задали внутри if;

    отличия var и let:

        var в тех же циклах представляет из себя одну единственную
        ячейку в памяти, в которой могут меняться значения по прохождению
        цикла, так что при использовании того же timeout в цикле, в итоге
        каждый из них выдаст лишь значение на момент активации;

        let в тех же циклах делает свою копию каждую итерацию для
        того же timeout;

    трудности:

        переменные поднимаются в начале кода, но var сразу берёт
        значение undefined, что может мешать. а let, в свою очередь,
        не даёт обратиться к себе до своего объявления в коде;

поиск элементов:

    document.querySelector('#id') - находит 1 элемент;

    document.querySelectorAll('#id') - находит группу элементов;

типы данных:

    number (число) - любые числа (даже дробные), а так же включает в себя
    спецзначения: Infinity и NaN;

    bigInt - для работы с очень большими целыми числами, которые не влезают
    в обычный number (для той же криптографии, или работы с финансами);

    string (строка) - текст в кавычках (одинарных, двойных или обратных),
    для хранения имён, сообщений, HTML-кода;

    boolean (логический тип) - только 2 значения (для условий или флагов,
    например, isLoggedIn);

    null - специальное значение, означающее ничего (например при намеренном
    обнулении переменной);

    undefined - автоматически присваивается переменной, если её не дели значение
    (для проверки, была ли инициализована переменная);

    symbol (символ) - уникальный идентификатор (например, для создания скрытых
    или уникальных ключей в объектах);

    ссылочный тип:

    object - самый мощный тип. позволяет хранить коллекции данных и более
    сложные сущности. включает в себя: массивы (arrays),
    функции (function), даты (date);

кавычки:

    '' и "" нужны для текста. всё, что находится вних читается буквально;

    `` - они могут делать интерполяцию строк, тут внутри вывода можно
    вставлять какие-либо значения следующим боразом:
        ${переменная или выражение};

в js есть 2 способа задания функций:

    обычный: function(event) {...};

    стрелочный: (event) => {...};

DOM - (Document Object Model) - дерево объектов HTML. все теги выстраиваются в
дерево, где каждый тег - узел. с каждым из них можно работать через тот же js, но
это весьма громоздко, так что есть Virtual DOM, а точнее VNode - это его копия,
находящаяся в памяти JS. при обновлении какого-либо маленького элемента, DOM начнёт
перерассчитывать всё так или иначе связанное, что ресурсоёмко. как VNode решает
 эту проблему:

    - копия в памяти: при любом изменении фреймворк сперва создаёт новую версию
    виртуального дерева;
    - Diffing (сравнение): сравнивает новую виртуальную копию со старой, чтобы
    найти конкретные отличия;
    - Reconciliation (точечное обновление): только после этого фреймворк идёт
    в реальный DOM и меняет там исключительно то, что действительно изменилось,
    не трогая остального;

основные отличия:

    DOM:
        - реальное представление страницы;
        - медленный при частых обновлениях;
        - тратит много ресурсов;
        - обновляется сразу весь узел;

    VNode:
        - обычный JS-объект (копия);
        - очень быстрый в памяти;
        - лёгкий и экономный;
        - обновляет только разницу;

ссылки на объекты:

    с помощью document.querySelector('') можно обращаться к конкретным тегам
    в HTML. в них есть множество параметров, которые можно редактировать,
    среди которых перезапись следующих видов:

        - .textContent - заменяет весь текст внутри тега, уничтожая даже и
        другие теги (если есть дочерние);
        - .innerHTML - если нужно добавить не просто текст;
        - .style - можно добавить изменения в CSS;

для анимаций можно использовать один из двух вариантов:

    setInterval - позволяет задавать изменения по временным промежуткам;

    requestAnimationFrame - некий золотой стандарт анимации, НО есть
    "подводные камни":

        зависимость от обновления частоты монитора (fps):
            у кого-то 60, у кого-то 144 и т.п. fps на мониторе, отчего
            будет разная скорость анимации. для исправления этого можно
            высчитывать дельту, но это морочно;
        
        остановка в фоновых вкладках:
            если переключиться на другую вкладку браузера, то эта функция
            ставится на паузу. после возвращения, анимация продолжится, отчего
            нужно делать счётчик времени и относительно него править анимацию;
