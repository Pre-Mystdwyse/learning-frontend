тернарный оператор:

    можно превратить это:

        let role;
        if (age >= 18) {
            role = 'adult';
        }
        elsr {
            role = 'child';
        }

    в это:

        let role = age >= 18 ? 'adult' : 'child';

    если нужно выполнить действие только при истинном условии, то из этого:

        if (isLoggedIn) {
            showDashboard();
        }

    в это:

        isLoggedIn && showDashboard();

    можно заменить проверку на существование значения. из этого:

        let name;
        if (username) {
            name = username;
        }
        else {
            name = 'Guest';
        }

    на это:

        let name = username || 'Guest';

    есть ещё оператор нулевого слияния. похож на ||, но срабатывает только тогда,
    когда левое значение строго равно null или undifined. полезно, если 0 или
    пустая строка "" считаются валидными значениями:

        //если totalCount равен 0, count останется 0
        //если бы использовали ||, count стал бы 10
        let count = totalCount ?? 10;

память:

    память разделена на 2 основные части:
        - stack - для быстрого хранения фиксированных значений
        (включая примитивы и адреса ссылок);
        - heap - для больших, динамических объектов;

    у примитивов по определению не может быть методов (функций);

    более детальный разбор иммутабельности:
        let user = { name" 'Карбел' };
        в куче создаётся объект { name: 'Карбел' } (например, по адресу #001);
        в стеке создаётся переменная user, и в неё записывается сам адрес #001;
        если сделать let admin = user;, то в стеке создастся переменная admin со
        значением #001. оба ключа ведут к одному сейфу в куче. изменив admin.name,
        будет изменено и user.name. в React так делать КАТЕГОРИЧЕСКИ НЕЛЬЗЯ, потому
        что React сравнивает объекты по их ссылкам. если адрес (#001) не зименился,
        React решит, что внутри объекта ничего не поменялось, и не обновит интерфейс
        на экране;
        чтобы React увидел изменения, нужно передать ему копию объекта с новым
        адресом в памяти. копирование бывает 2-ух видов:

            поверхностное копирование:
                обычно используют Spread-оператор(...). он создаёт новый объект
                в памяти, но копирует свойства первого уровня:
                    const = original = {
                        name: 'Карбел',
                        skills: ["магия", "внимательность"]
                    };

                    const copy = { ...original };
                    //original !== copy (ссылки на объекты разные и this is хорошо)

                    copy.name = 'Маркез'; //original.name остаётся 'Карбел'
                    copy.skills.push('знание легенд');
                    //а вот тут изменились оба объекта

                почему так вышло? Spread-оператор скопировал строку name по
                значению (создав новую), но внутренний массив skills он скопировал
                по ссылке. в итоге original.skills и copy.skills указывают на
                один и тот же массив в куче;

            глубокое копирование:
                глубокое копирование дублирует абсолютно всё: и сам объект, и все
                вложенные в него объекты и массивы. у них будут совершенно новые
                адреса в памяти;
                вот основные способы сделать глубокое копирование:

                    современный стандарт - structuredClone():
                        это встроенный в js метод. он идеально подходит для
                        глубокого копирования:
                            const original = {
                                name: 'Карбел',
                                skills: [ 'магия', 'внимательность' ]
                            };

                            const deepCopy = structuredClone(original);

                            deepCopy.skills.push('знание легенд');

                            console.log(original.skills);
                            //[ 'магия', 'внимательность' ]
                            console.log(deepCopy.skills);
                            //[ 'магия', 'внимательность', 'знание легенд' ]

                    старый "хак" - JSON.parse(JSON.stringify()):
                        раньше, до появления structuredClone, все использовали
                        этот трюк. мы превращаем объект в строку, а потом из строки
                        создаём новый объект:
                            const deepCopy = JSON.parse(JSON.stringify());
                        - минусы: этот способ ломается, если внутри есть функции,
                        методы, даты (Date), Map, Set иди undefined свойства. они
                        просто исчезнут или превратятся в кашу. для простых объектов
                        с текстом и числами использовать можно, но в 2026 году лучше
                        отдавать предпочтение structuredClone();

                важное правило для React-разработки:

                    в React нужно постоянно обновлять "стейт" (состояние компонента),
                    поэтому нужно запомнить правило: НИКОГДА НЕ МУТИРОВАТЬ СТАРЫЙ СТЕЙТ
                    НАПРЯМУЮ. вместо этого всегда создавать копию (поверхностную или
                    глубокую), менять данные в ней и отдавать React эту новую копию;
