position:
    до этого момента элементы шли "в потоке" - один за другим. но что, если
    понадобится повесить значок "New" в углу карточки, или закрепить меню
    сверху, чтобы оно не уезжало при скролее? для этого нужно свойство
    position;

    1. position: relative (относительное):
        элемент остаётся в потоке, но можно его чуть-чуть подвинуть
        (top, bottom, left, right) относительно того места, где
        он должен быть;

        - главная фишка: он становится якорем для детей с absolute;

    2. position: absolute (абсолютное):
        элемент вылетает из потока. другие элементы его больше не видят
        и занимают его место;

        - он позиционируется относитеьлно ближайшего родителя, у которого
        стоит position: relative (или absolute/fixed);

        - если у родителя ничего нет - он летит к краю всего окна браузера;

        - использование: иконки в углах блоков, всплывающие подсказки;

    3. position: fixed (фиксированное):
        элемент "приклеивается" к экрану. скроллишь ты или нет - он всегда
        на одном месте;

        - использование: навигация вверху страницы (header) или кнопка
        "наверх";

синтаксис box-shadow: [inset] [смещение x] [смещение y] [радий размытия]
[радий растяжения] [цвет];

    1. inset (необязательно) - если указано, тень становится внутренней;

    2. offset-x (горизонтальное смещение) - обязательно. положительное
    значение - тень справа, отрицательное - слева;

    3. offset-y (вертикальное смещение) - обязательно. положительное -
    тень снизу, отрицательное - сверху;

    4. blur-radius (радиус размытия) - необязательно. чем больше значение,
    тем сильнее размыта тень;

    5. spread-radius (радиус растяжения) - необязательно. положительное -
    тень увеличивается, отрицательное - уменьшается;

    6. color (цвет) - ну, как бы надо;

    несколько теней: box-shadow: 3px 3px 5px grey, -3px -3px 5px yellow;
    первая тень - поверх остальных;

    если у элемента есть border-radius, то тень будет повторять их форму;

подробнее про position: relative:
    пример: в очереди стоят 3 человека. каждый занимает своё место;
        - если второй человек - static (по умолчанию), он стоит ровно
        там, куда его поставила очередь;
        - если дать ему position: relative; top: 10px;, он физически
        смещается на 10 пикселей вниз;
        - НО! первый и третий человек в очереди не замечают этого. для
        них второй человек всё ещё занимает своё старое место. очередь
        не смыкается, никто не двигается;
        - это и есть относительное смещение: элемент сдвинулся только
        визуально, сохранив за собой своё законное пространство в потоке;

псевдоэлементы:

    ::before и ::after - это виртуальные дети элемента, которые создаются
    чисто через CSS:
    
        - ::before - появляется перед контентом элемента;
        - ::after - появляется после контента;

    главное правило: атрибут content:

        псевдоэлемент не появится, если не написать ему свойство
        content: "";. даже если оно пустое;
    
    как использовать с позиционированием:

        можно сделать так, чтобы (в этом случае) у заголовка или
        карточки появилась неоновая полоска снизу. пример:

            #character-profile::before {
                content: "";    <!-- ОБЯЗАТЕЛЬНАЯ СТРОКА -->
                position: absolute;
                top: -5px;
                left: 10%;
                width: 80%;
                height: 2px;
                background: #83da8e;
                box-shadow: 0 0 10px #83da9e;
            }

pointer-events: none нужно для того, чтобы сделать эту форму прозрачной
для мыши, ибо без этого параметра она, т.к. накладывается поверх основного
контента, то забирает на себя взаимодействия с мышью;

странное поведение position: absolute; top: 0; left: 0; с border:

    когда задаются такие параметры элементу, он приклеивается к внутреннему
    краю рамки родителя (к началу padding-box);

    в данном случае рамка профиля 2px. значит ::after начинается на 2px
    правее и ниже, чем внешний край основной карточки. а т.к. его width
    и height равны 100% (внутренней части карточки), он кажется смещённым;

    исправление:

        просто пододвинуть псевдоэлемент на 2 пикселя назад, чтобы он
        перекрыл рамку родителя. в этом случае вместо width и height
        лучше использовать 4 точки привязки: top, left, bottom, right;

gap:
    это современное свойство, задающее промежутки (отступы) между строками
    и колонками внутри контейнеров flexbox, grid или многоколоночного
    макета. оно заменяет сложные комбинации margin, предотвращая появление
    лишних отступов у крайних элементов;

    row-gap - между рядами;
    column-gap - между колонками;
