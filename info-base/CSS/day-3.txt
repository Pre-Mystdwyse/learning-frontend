3 способа подключения CSS:

    1. инлайново - style="..." - плохой тон;
    2. тег <style> внутри <head> - удобно для маленьких правок, но не для полного сайта;
    3. внешний файл .css - золотой стандарт. создаём отдельный файл и подключаем его
    одной строкой в HTML. это позволяет менять дизайн всего сайта, не заходя в HTML-код;

анатомия CSS-правила:

    селектор {
        свойство: значение;
    }

    - селектор - это кого мы красим (например, h1);
    - свойство - это что именно крачим (например, color - цвет текста);
    - значение - это в какой цвет (например, violet);

базовые селекторы:

    - по тегу p {...} - применится ко всем абзацам;
    - по классу .my-class {...} - применится только ко всем элементам с class="my-class"
    (самый частый способ);
    - по ID #my-id {...} - применится только к одному уникальному элементу;

одинаковые стили можно применить ко множеству тегов, классов и id обозначив их через
запятую: h1, p, .i2pupuye, #some-id {...};

стили можно применить к тегам, классам и id:
    Специфичность:
        тег = 1 балл;
        класс = 10 баллов;
        ID = 100 баллов;
    (браузер суммирует баллы и применяет тот стиль, который "тяжелее")

Блочная модель (Block Model):

    в CSS всё, абсолютно всё - это коробка. даже если элемент выглядит как
    круг или текст - для браузера это прямоугльный контейнер;

    Из чего состоит "коробка" (из нутри наружу):

        1. Content (контент) - сама картинка, текст или видео. у него есть
         ширина (width) и высота (height);
        
        2. Padding (внутренний отступ) - расстояние от контента до границы коробки;

        3. Border (граница/рамка) - та самая линия вокруг контента и паддинга. она
        может быть толстой, тонкой, пунктирной;

        4. Margin (внешний отступ) - расстояние от границы одной коробки до другой. это
        "социальная дистанция" между элементами;

        Важная настройка:
        по умолчанию в CSS паддинги и бордеры увеличивают размер коробки. т.е. если задать
        width: 100 и добавить padding: 20px, то коробка станет 140px. это дико неудобно. поэтому
        в начале любого проекта стоит писать строку, заставляющую считать размер адекватно:

        *{
            box-sizing: border-box;
        }

        эта строка говорит: "пусть ширина всегда остаётся такой, каковой она была указана,
        а паддинги и бордеры влезают внутрь неё;

коробка - это всё вместе: content, border, padding и margin;

приписка auto в margin - команда взять всё свободное место справа и слева и
разделить его поровну;
важное условие: чтобы auto сработало, у карточки должна быть задана ширина
(например width: 600px или max-width: 80%). если ширина не задана, блочный
элемент (selection) по умолчанию растянется на все 100% и делить браузеру
будет нечего - auto превратится в 0;

фон закрашивает content и padding, но не трогает margin;

в CSS есть 3 основных способа задать цвет:

    1. ключевые слова: violet, silver, transpanent (грубо и мало вариантов);

    2. HEX-коды: #2c094b - это шеснадцатиричная система (RGB). первые 2 цифры -
    красный, вторые 2 - зелёный, третьи - синий. самый частый формат;

    3. RGBA: rgba(158, 15, 241, 0.5) - то же самое, но четвёртое число -
    это прозрачность (от 0 до 1). идеально для наложений и магических эффектов;

единицы измерения (px vs rem vs %):

    - px (пиксели): жёсткая привязка. 20px - это всегда 20px. плохо для мобилок;
    - % (проценты): зависимость от родителя. width: 50% - это половина ширины контейнера;
    - rem - король современной вёрстки. 1rem обычно равен 16px (стандартный размер
    шрифта браузера). если пользователь в настройках браузера увеличит шрифт, то сайт
    "поплывёт" вслед за ним. не ломаясь (стоит использовать rem для текстов и отступов);

есть возможность выбрать первый тег внутри другого тега -
для этого используется псевдокласс, например:

    чтобы выбрать первый абзац <p> внутри <article> вне зависимости от
    того, есть ли перед ним что-то ещё, лучше сделать так:

        article p:first-of-type {
            /*стили*/
        }

        НО! если поставить пробел после p, то будет искаться первый
        элемент ВНУТРИ p;

есть как rem, так и em:

    em - единица, которая зависит от размера шрифта родителя;

    rem (Root EM) - это единица, которая всегда привязана ТОЛЬКО К КОРНЮ,
    т.е. к тегу <html>;
    можно сменить это стартовое значение, задав своё font-size конкретно
    для тега html;
    из этого следует, что изменение в body не влияет на rem.

псевдоклассы и селекторы:

    псевдоклассы (состояния):

        :hoover - наведение мышкой;
        :active - момент нажатия;
        :focus - когда в инпут поставили курсор (критично для фром);
        :nth-child(n) - позволяет выбрать, например, каждый второй
        элемент списка или только пятый;

    комбинаторы (связи):

        например:
        1. article p - вложенный селектор, который находит все p внутри article;

        2. article > p - выберет только те p, которые являются прямыми детьми
        article (если p лежит внутри div, который лежит внутри article - он
        не выберется);

        3. h2 + p - выберет только тот p, который идёт сразу после h2 (удобно
        для эпиграфов под заголовками);

        4. input[type="text"] - выберет только текстовые инпуты, не трогая чекбоксы;
