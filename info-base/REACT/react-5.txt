разница между => {...} и => ({...}):
    в js это называется явным и неявным возвратом стрелочной функции. зависит тут всё
    строго от наличия фигурных скобок;

    1. квадратные скобки или просто текст:
        если после стрелки нет фигурных скобок, функция выполняет код в этой строке
        и автоматически возвращает результат
        const add = (a, b) => a + b; //вернёт сумму чисел

    2. передача объектов?
        в js фигурные скобки имеют двойной смысл: они либо означают тело функции, либо
        объект. поэтому если нужно вывести объект, необходимо дважды писать фигурные скобки:
            const getHero = () => {{ name: 'Карбел' }}; //вернёт объект
        
    правила просты: есил нужно вернуть 1 объект - нужно писать автовозврат => ({...});
    если нужно что-либо ещё делать внутри этой функции (даже банальный console.log()),
    то тут нужно писать {...; returt(...);}


в create((set) => ...) в set записывается системная функция самого zustand. когда zustand
запускает create(...), он под капотом создаёт скрытый механизм отслеживания активности. и в
качестве 1-го аргумента он передаёт пульт управления этим механизмом - функцию set;

каждый раз, когда вызывается set(), zustand берёт объект, который ему передали, склеивает
его с текущим состоянием и заставляет реакт перерисовать компоненты;

setHero: (newHeroOrFn) => set((state) => {...}):
    1. setHero: (newHeroOrFn) => ... - это функция, которая будет вызываться в магазине;
    в неё передаются данные;
    2. set(...) - внутри этой функции вызывается пульт управление zustand;
    3. (state) => {...} - функции set из zustand можно передать либо готовый объект
    (например, set({ gold: 100 })), либо другую стрелочную функцию;

    если передать в set функцию (state) => {...}, то zustand понимает это так:
    "разработчик хочет сперва посмотреть, что сейчас лежит в сторе, прежде чем
    обновлять данные";

    zustand берёт самое актуальное в эту наносекунду состояние памяти и сам подставляет его
    в аргумент, который мы назвали state;

    внутри фигурных скобок можно написать:
        - state.hero - чтобы посмотреть текущего героя;
        - state.history - чтобы посмотреть массив истории;

    после проведения вычислений, в конце нужно писать return { hero: ... }. то, что
    возвращено из этой внутренней функции, функция set забирает и вносит в глобальный стейт;
