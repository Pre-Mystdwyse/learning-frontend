в React метод .map() используется для превращения массивов данных в списки элементов
интерфейса (JSX). это обычный инструмент из языка JS, который берёт каждый элемент из
списка (например, имена) и делает из него видимый на экране кусочек (например, тег списка);

как работает?
метод .map() берёт старый массив, пробегается по каждому его элементу, изменяет его по
заданному правилу и создаёт из него новый массив;

обычно в React внутри .map() пишут код, который превращает текст или цифры в HTML-теги (JSX);

пример кода, в котором есть список фруктов и нужно показать их на экране в виде списка:

    function FruitsList() {
        //1. есть массив сданными
        const fruits = ['яблоко', 'банан', 'апельсин'];

        return (
            <ul>
                {/* 2. превращаем каждый фрукт в тег <li> */}
                {fruits.map((fruit, index) => (
                    <li key={index}>{fruit}</li>
                ))}
            </ul>
        );
    }

зачем нужен key?
когда используется .map(), React всегда просит добавить специальный атрибут key для каждого
элемента списка:
    - зачем нужно: ключ помогает React понимать, какие именно элементы в списке изменились,
    добавились или удалились;
    - что использовать: лучше всего использовать уникальный ID из базы данных (например,
    user.id), а не просто порядковый номер элемента;

главные плюсы .map() в React:
    - автоматизация: не нужно вручную копировать теги для 100 товаров или пользователей;
    - чистота кода: код получается коротким и легко читается;
    - обновления: если массив данных изменится, React сам быстро перерисует список на экране;

внутри .map() создаётся функция. метод .map() сам передаёт в эту функию 3 вещи по очереди:
    1. сам текущий элемент массива;
    2. порядковый номер элемента;
    3. весь исходный массив;

чтобы сделать на основе объекта массив, нужно использовать Object.entries();
    - Object - это встроенное в JS системное слово (главный объект). оно даёт доступ к
    разным полезным инструментам для работы со всеми объектами;
    - .entries(...) - это метод, который берёт переданный в него и разбирает его на
    пары "ключ-значение";

если в .map() подаётся уже структурированный массив, в котором уже есть ключ и значение, то
нужно писать вывод в .map() именно так, как расположены относительно друг друга эти ключ и
значение;
в случае с const itemsArray = Object.entries(ITEMS); записываются сперва ключи, а затем значения,
отчего в .map() будут передаваться сперва ключи, а потом значения;

по правилам JS, если аргумент функции выглядит как квадратные скобки, то их обязательно нужно
обернуть в круглые;

в стрелочных функциях есть 2 варианта как вернуть результат:

    - с круглыми скобками:
    это называется неявный возврат. не нужно писать слово return. всё, что находится внутри
    этих скобок автоматически выводится наружу:
        .map(([key, itemData]) => (
            <ShopItem key={key} item={itemData} />
        ))
    
    - с фигурными скобками:
    эти скобки открывают тело функции. внутри них можно писать обычный код, но из-за этого
    JS не возвращает ничего сам. обязательно прописывать return:
        .map(([key, itemData]) => {
            //тут можно написать чё-то ещё

            return (
                <ShopItem key={key} item={itemData} />
            );
        })

useEffect (хук побочных эффектов):
    это функция, которая говорит: "выполняй этот код каждый раз, когда
    меняется эта переменная":
        useEffect(() => {
            saveHero(hero); //сохраняем в LocalStorage
            console.log('Данные героя синхронизированны!');
        }, [hero]); //[hero] - это список зависимостей. код срабатывает только
        //при изменении hero
