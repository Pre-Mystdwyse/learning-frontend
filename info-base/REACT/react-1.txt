при работе с чистым JS - это императивный подход (когда я говорю браузеру: "сделай это,
потом то, затем вон то");
React работает по-другому - декларативно. я более не командую браузером. я просто
описываю : "если золото = X, то на экране должна быть такая-то строка. если в инвентаре
Y предметов, то нарисуй их иконки";

главная формула React:

    UI = f(State)
    (интерфейс - это функция от состояния)

    меняю данные (state) - React сам мгновенно перерисовывает нужные части экрана. в моём
    случае, uiUpdater.js больше не нужен, React возьмёт эту рутину на себя;

npm install react react-dom
эта команда нужна для установки самого React;

npm install -D @vitejs/plugin-react
эта команда - официальный плагин, который учит vite работать с синтаксисом React (JSX/TSX),
обеспечивает автоматическое обновление компонентов в браузере без перезагрузки страницы
(fast refresh) и оптимизирует код;

в React HTML пишется внутри кода;
при этом вместо class нужно писать className чтобы не было возможных проблем (class -
зарезервировано js, поэтому в React выбрали calssName);

обратботка событий:

    вместо addEventListener мы пишем onClick={...} прямо на кнопке (в HTML);
    

обязательно (вроде как) для каждого проекта на React:

    import React from "react"
    import ReactDOM from 'react-dom/client'
    import App from './App'
    import './CSS/style.css'

    ReactDOM.createRoot(document.getElementById('root')).render(
        <React.StrictMode>
            <App />
        </React.StrictMode>
    )

    ReactDOM.createRoot(document.getElementById('root')) ссылается на
    div в HTML с id="root" - это становится корнем для вставок;

    <React.StrictMode> нужен для помощи с разработкой;

export default App; позволяет указать, что это главный файл, а так же сократить
его импорт:
    вместо import { App } from './App'; можно писать import App from './App';


когда я делаю константу const [hero, setHero] = useState(getHero());, то
React создаёт внутри себя состояние. константа hero - это снимок (snapshot)
данных в текущий момент. если написать hero.gold -= 500;, то я изменю значение
в памяти, но React не узнает об этом, ибо он не следит за каждым свойством
объекта. он просто хранит адрес этого объекта в памяти;
setHero(updateHero); - здесь я передаю новую ссылку и React видит, что ссылка
изменилась, поэтому и перерисовывает интерфейс;

StrictMode:
    React специально запускает функции дважды в режиме разработки: если функция
    делает что-то "грязное" (например, лезет в DOM или меняет глобальную
    переменную), то при двойном запуске это вызовет баг, который будет сразу
    заметно. это некий тренажёр для написания чистого кода;
