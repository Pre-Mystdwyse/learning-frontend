Контейнер <form>:
    некий логический блок;
    
    Атрибуты:
        action - адрес, куда отправится содержимое;
        method - выбор отправки либо get-запросом, либо post-запросом;

подробнее про get и post:
    
    GET - получение:
        при отправке формы методом get, все данные из полей ввода приклеиваются
        прямо к адресу сайта (URL);

        пример: mysite.com/search?hero=Karbel&level=15
        (ввели имя персонажа и его уровень)
        особенности:
            - видимость - все данные видны в url;
            - ограничения - длина url ограничена (около 2к символов);
            - безопасность - ага, щас. нету. не использовать для паролей и личных данных;
            - кэширование - браузер запоминает такие ссылки. удобно для поиска или фильтров
            (можно скинуть ссылку конкретно на отфильтрованный список товаров).

    POST - отправка:
        данные упаковываются внутрь тела HTTP-запроса. они не видны в адресе.

        пример: в url mysite.com/update-hero, а данные едут за кулисами;
        особенности:
            - скрытность - данные не отображаются в истории браузера или url;
            - объём - можно передавать огромные объёмы данных;
            - безопасность - лучше, ибо данные сложнее перехватить "случайным взглядом",
            но для полной защиты нужен HTTPS-шифр;
            - кэширование - браузер не кэширует такие запросы. при нажатии назад после
            отправки POST-формы, браузер спросит "Вы уверены, что хотите отправить
            данные ещё раз?".

    когда что выбирать?
        - GET - для поиска, фильтрации, навигации (просто поиск, просмотр);
        - POST - для регистрации, смены пароля, сохранения персонажа, отправки сообщений,
        когда мы что-то меняем или создаём на сервере.
        
<input> - самый частоиспользуемый тег в формах;
нет закрывающего тега;
его поведение полностью зависит от атрибута type.

магия в атрибуте type:

    1. Текстовые:
        type="text" - короткая строка (имя героя, название предмета);
        type="password" - то же самое, но вместо букв - точки, для безопасности (скрывает символы);
        type="email" - проверяет есть ли в тексте @ и точка. на смартфоне сразу даёт клавиатуру с @;
        type="tel" - для номеров телефонов. вызывает цифровую клавиатуру на смартфонах;
        type="url" - проверяет корректность ссылки;

    2. Выбор и переключатели:
        type="checkbox" - галочки (множественный выбор);
        type="radio" - точка (единичный выбор);
        type="color" - открывает системную палитру для выбора цвета;

    3. Числа и диапазоны:
        type="number" - поле для ввода чисел со стрелочками (><), а так же можно добавить
        дополнительные атрибуты, чтобы сделать удобнее: min, max, step;
        type="range" - ползунок;

    4. Дата и время:
        type="date" - выбор дня, месяца, года;
        type="time" - выбор времени;
        type="datetime-local" - выбор полного времени (два предыдущих в одном);
        type="month" / type="week" - более узкие выборки;

    5. Служебные:
        type="file" - кнопка "Обзор" для загрузки файлов с ПК;
        type="hidden" = поле, которое пользователь не видит, но оно отправляется на сервер
        (например ID сессии);
        type="submit" - кнопка, которая отправляет форму;
        type="button" - просто кнопка, которая сама по себе ничего не делает до тех пор, пока
        не сделать ей логику на JS.

UX-база - <label>:
    текст рядом с полем ввода должен быть обёрнут в <label>. зачем?
    - для удобства, чтобы кликабельным было не только поле, но и текст;
    - чтобы браузер понимал, что текст относится к этому полю

    как связывается?
    по принципу "паспортных данных". нужно дать инпуту уникальный id, а лейблу сообщаем:
    "ты работаешь ДЛЯ (for) этого id":

    <label for="int-points">Интеллект: </label>
    <input type="number" id="int-points">

Разница Checkbox и Radio

    - type="checkbox" - галочка. можно выбрать СКОЛЬКО УГОДНО вариантов;
    - type="radio" - переключатель. можно выбрать ТОЛЬКО 1 вариант.

    ВАЖНО!
        чтобы браузер понял, что кнопки относятся к одной группе и нельзя выбрать
        2 сразу, то им нужно дать одинаковый атрибут name:

        <input type="radio" name="difficulty" id='easy">
        <label for="easy">Легко</label>

        <input type="radio" name="difficulty" id="hard">
        <label for="hard">Хардкор</label>

        если у них будут разные name, то можно будет выбрать хоть оба варианта
        одновременно, но тогда и потеряется смысл radio.

    атрибуты-подсказки:
        - placeholder - серый текст внутри поля, который исчезает, при заполнении поля;
        - value - значение по умолчанию;
        - required - форма не отправится, пока полне не будет заполнено;
        - readonly и disable - поле видно, но в нём нельзя ничего менять.

Выпадающие списки, текстовые области и кнопки:

    1. <select> и <option>: выпадающее меню:
        если нужно сделать выбор из 10+ вариантов (например выбор города), то радио-кнопки -
        не вариант. тут нужен выпадающий список:
            <select> - это сам контейнер-раскладушка;
            <option> - это пункты внутри него.
        
        пример:

        <label for="class-select">Выберите класс</label>
        <select id="class-select" name="hero-class">
            <option value="warrior">Воин</option>
            <option value="mage" selected>Маг</option>
            <!-- selected делает этот пункт выбранным по умолчанию -->
            <option value="rogue">Плут</option>
        </select>

        - важный атрибут value: пользователь видит слово, находящееся в option, но серверу
        отправляется техническое слово, например, mage. это позволяет менять текст на экране
        (как например, перевод на другой язык), не ломая логику программы;

        - <optgroup> - если вариантов очент много, их можно группировать подзаголовками
        внутри списка. пример реализации:

        <select name="hero-class">
            <optgroup label="Ближний бой">
                <option value="warrior">Воин</option>
                <option value="paladin">Паладин</option>
            </optgroup>
            <optgroup label="Магия">
                <option value="mage">Маг</option>
                <option value="warlock">Чернокнижник</option>
            </optgroup>
        </select>

        визуально в браузере названия "Ближний бой" и "Магия" будут выделены жирным,
        а варианты под ними - немного сдвинуты вправо;

    2. <textarea> - большие тексты:
        когда нужно написать не просто имя, а целую биографию или комментарий, обычный
        <input> не подходит (он однострочный);

        - <textarea> - это поле, которое можно растягивать;
        - особенность - в отличие от input, этот тег обязательно нужно закрывать;
        - атрибуты: rows и cols - задают начальный размер поля в строках и символах.

        пример:

            <label for="bio">Биография</label>
            <textarea id="bio" name="story" rows="5" cols="33">
            Здесь можно написать текст, который будет внутри по умолчанию.
            </textarea>

    3. <button> vs <input type="submit">:
        в HTML есть 2 способа сделать кнопку для отправки формы:

        I. старый способ:
            - пишется так: <input type="submit" value="Отправить">;
            - минус: нет возможности положить внутрь такой кнопки картинку или иконку. она
            умеет отображать только простой текст из атрибута value;
        
        II. современный способ:
            - пишется так: <button type="submit">Отправить</button>;
            - плюс: это парный тег. внутрь него можно положить что угодно: <strong>, <img>,
            <span>. это делает огромную свободу для дизайна.

        Типы кнопок:
        - type="submit" - отправляет данные формы (стоит по умолчанию);
        - type="reset" - очищает все поля формы (вряд ли кому-то понравится случайное удаление
        введённых данных);
        - type="button" - "пустая" кнопка. она ничего не делает, пока не напишем скрипт на JS.

    
группировка полей <fieldset> и <legend>:
    если форма очень большая (например, настройка профиля), её стоит разбить на логические блоки.

    - <fieldset> - рисует рамку вокруг группы полей;
    - <legend> - вставляет заголовок прямо в эту рамку;

    пример:

        <fieldset>
            <legend>Основные параметры</legend>
            <!-- Тут инпуты для имени и возраста, к примеру -->
        </fieldset>
